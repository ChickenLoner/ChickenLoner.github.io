Data Loading Pattern
JS — detail page (fetch own metadata)
const [meta, setMeta] = useState(null);
useEffect(() => {
    fetch('/data/reviews.json')
        .then(r => r.json())
        .then(data => {
            const slug = window.location.pathname.replace(/\/$/, '');
            setMeta(data.find(r => r.url === slug + '/index.html'));
        })
        .catch(() => {});
}, []);

Use meta?.rating, meta?.difficulty, meta?.date etc. with — as fallback. Never hardcode rating or difficulty in HTML.

themes/soc-theme.js

Synchronous IIFE — load without defer. Applies data-theme to <html> before any render, preventing flash of wrong theme.

  • Reads localStorage['soc-theme'] on load (default: 'dark')
  • Exposes window.SocTheme = { toggle, get, set }
  • set(theme) persists to localStorage, sets data-theme on <html>, dispatches soc-theme-change CustomEvent on document
  • React components listen to soc-theme-change to update toggle icon and (on index.html) to remove the Tailwind .dark class
  • Does not use prefers-color-scheme — explicit user choice only
KeyValue
localStorage keysoc-theme
Defaultdark
HTML attributedata-theme="light" on <html>
CSS selectorhtml[data-theme="light"] — specificity 0,1,0,1 beats inline :root 0,1,0,0
themes/soc-nav.js

Injected globally via <script src="/themes/soc-nav.js" defer>.

  • On DOM ready: injects fixed overlay #soc-nav-overlay (progress bar + ring spinner + "UPLINK CONNECTING_" text)
  • On any internal link click: shows overlay, waits 420ms, then navigates
  • On page load: fades overlay out
  • Uses sessionStorage to distinguish first-load vs navigation-load
  • Does not affect: external links (target="_blank"), anchor links (#id), mailto:, tel:
themes/soc.css
  • CSS variable definitions — canonical dark palette in :root
  • html[data-theme="light"] block — overrides all 15 --soc-* vars to light values
  • [data-theme="light"] component overrides — .soc-tb, body background, severity badges, section icons, callouts, cover overlays, card shadows, .soc-theme-btn
  • --soc-*-tint-* CSS vars (cy/am/vi/rd/gn × lo/md/hi/bdr levels, dark + light values)
  • Tailwind .dark class overrides → SOC palette
  • ~140 Tailwind utility equivalents (icon sizes, typography, spacing, layout, colors, borders, accessibility)
  • Global dark scrollbar styling
  • All .res-* and .ir-* component classes

Note: Detail pages also declare :root inline for self-containment. soc.css is the canonical reference. The html[data-theme="light"] selector (specificity 0,1,0,1) beats inline :root (0,1,0,0), so light mode overrides work on all pages without per-page edits.

Adding New Content

Use Claude Code skills — /add-cert, /create-review, /new-research, /new-ir-report automate these workflows.

New Certification Review
  1. Add entry to data/reviews.json
  2. Add cert to data/certifications.json
  3. Drop badge PNG → assets/badges/<slug>.png
  4. Create reviews/<slug>/index.html using reviews/psap/index.html as template
  5. Destructure from window.SocComponents — don't redefine shared components
New Research Article
  1. Add entry to data/research.json
  2. Add cover → assets/research/<slug>/cover.png
  3. Create research/<slug>/index.html using research/anydesk-forensics-windows/index.html as template
New IR Report
  1. Add entry to data/ir-reports.json
  2. Add cover → assets/labs/<slug>.jpg
  3. Create ir-reports/<slug>/index.html using ir-reports/htb-tinsel-trace-1/index.html as template

Conventions & Constraints
  • Dark/light toggle. soc-theme.js manages theme via data-theme on <html>. Default: dark. Persisted in localStorage['soc-theme']. ThemeToggle component in every topbar.
  • No backend. Pure static HTML/JS/CSS + JSON.
  • No router library. Filesystem-based routing.
  • Self-contained CSS. :root CSS variables declared inline on every page — pages render correctly if /themes/soc.css fails.
  • OG/Twitter meta required on all pages. Always include when creating new pages.
  • JetBrains Mono everywhere. Monospace font is core to the SOC aesthetic.
  • Shared components via soc-components.js. Icon, SocClock, Section, Fig, CodeBlock, Img, TipList, B, A, Code, dedent sourced from window.SocComponents. Per-page components (Callout, SubSection, ReportSection, etc.) remain inline.
  • No Tailwind CDN on detail pages. Utility equivalents live in soc.css.
  • Pinned CDN versions. Babel 7.29.2, Lucide 1.14.0, Tailwind 3.4.17.
  • Assets co-located by content type. /assets/research/[slug]/ for research, /assets/reports/[slug]/ for IR, /assets/badges/ for certs.
  • Favicon: /chicken0248.png on all pages.
